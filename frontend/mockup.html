Firefox Sync Setup
Authenticate with your identity provider to configure Firefox Sync with your self-hosted server.

This tool will guide you through authenticating with your OIDC provider and obtaining the Token Server URI needed to configure Firefox Sync.

How it works

  1. You sign in with your identity provider
  2. We verify your credentials with the Token Server
  3. You get the Token Server URI to paste into Firefox's about:config

Security

Authentication happens directly with your OIDC provider using PKCE to prevent code interception. No credentials are stored by this application — all session data is cleared when the tab closes.

Credential lifetime

HAWK credentials expire after 300 seconds. Firefox will automatically refresh them once configured.

Requested permissions

  • openid — Basic identity verification
  • profile — Your name and profile information
  • email — Your email address

Configure Firefox Sync

You must set the Token Server URI before signing into Firefox Sync.
  1. Open Firefox and type about:config in the address bar
  2. Accept the risk warning if prompted
  3. Search for identity.sync.tokenserver.uri
  4. Set the value to the Token Server URI shown above
  5. Sign into Firefox Sync normally using your identity provider credentials

Once configured, Firefox will automatically obtain and refresh credentials from the Token Server. The Token Server validates all requests to ensure security.

Re-authentication is only needed for testing. Firefox handles authentication automatically once configured.

You can try authenticating again. If the problem persists, check your OIDC provider and Token Server configuration.
